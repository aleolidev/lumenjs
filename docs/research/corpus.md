# Initial reference corpus

This seed corpus is intentionally diverse. It will be expanded and scored per
capability during the repository sweep.

## Fangames

| Reference | Initial reason for inclusion |
| --- | --- |
| Pokémon Reborn | Field effects, long-form narrative, difficulty, passwords, postgame. |
| Pokémon Rejuvenation | Quests, branching narrative, custom fields and progression systems. |
| Pokémon Insurgence | Variants, customization, challenge modes, bases, online features. |
| Pokémon Uranium | Original creatures and type, integrated challenges, trading and gifts. |
| Pokémon Infinite Fusion | Compositional creatures, community asset scale, configurable randomizer. |
| Pokémon Xenoverse | Original region and creatures, distinctive presentation and progression. |
| Pokémon Sage | Original-content production and cohesive regional design. |
| Pokémon Opalo | Large Spanish-language campaign and multi-region scope. |
| Pokémon Iberia / Añil | Spanish community reach, writing, localization, and creator workflow. |
| Pokémon Phoenix Rising | Presentation ambition and lessons from incomplete projects. |

## Creation platforms and adjacent systems

| Reference | Initial reason for inclusion |
| --- | --- |
| Pokémon Essentials | Dominant accessible fangame workflow, PBS data, events, plugins, debug. |
| PSDK / Pokémon Studio | Dedicated database editor, mapping integrations, localization, tooling. |
| RPG Maker XP | Event authoring and beginner accessibility; inherited constraints and debt. |
| pret decompositions | Source organization, reproducible builds, documentation, historical behavior. |
| Pokémon Showdown | Data-driven battle simulation, formats, validation, replays, online operation. |
| Tiled | Mature general map authoring and extensibility. |

## Coverage still required

- More non-English and region-specific communities.
- Mobile-first and browser-native fangames.
- Multiplayer and MMO-like projects.
- 2.5D and 3D experiments.
- Specialist battle, map, shader, asset, dialogue, and localization tools.
- Failed, abandoned, or rewritten engines with documented lessons.
- General JavaScript/WebGPU engines and repositories, evaluated separately from
  Pokémon-specific product references.
